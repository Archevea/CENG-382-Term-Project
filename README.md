# Class Reservation System  

---

## 1. Project Information and Objectives

### 1.1 Project Scope

The scope of this project is a **web-based classroom reservation system** designed for educational institutions. The system allows:

- **Instructors** to:
  - View classroom availability,
  - Request reservations via a calendar,
  - Provide feedback and ratings for classrooms.

- **Administrators** to:
  - Manage instructors and admin user accounts,
  - Review, approve, or reject reservation requests,
  - Maintain classroom and academic term information,
  - View system statistics.

- **System Features** include:
  - Integration with a public holiday API to block reservations on national holidays.

### 1.2 Users

- **Instructors**
  - Can request classroom reservations via an interactive calendar
  - Can view and track their reservations
  - Can give feedback and ratings for classrooms

- **Administrators**
  - Manage instructors and admin user accounts
  - Review, approve, or reject reservation requests
  - Maintain classroom details and academic terms
  - View reports and system statistics

- **System (Internal Use)**
  - Connects to a public holiday API to automatically block reservations on holidays

### 1.3 Objectives

- The system should provide password-protected access to registered users.
- It should support role-based access control (Admin and Instructor).
- Instructors should be able to submit reservation requests using a calendar interface.
- Admins should be able to approve, reject, or delete reservation requests.
- The system must integrate with a public holiday API to block reservations on official holidays.
- Instructors should be able to give feedback and rate classrooms.
- The admin dashboard should include usage statistics.
- Admins should be able to activate/deactivate user accounts.
